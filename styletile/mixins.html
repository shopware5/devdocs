_mixins

The _mixins directory contains all styled mixin-elements of shopware.
Mixins can be a huge help when creating cross-browser styles or even just shorthands. Use any of the following mixins in its purest form (like .border-radius()) to get the default values with cross-browser support.

view file

Unitize

Unitize is our own tool to calculation relative values like em and rem from pixel values.
It's a powerful tool, see here why: 

// @property {String}: The property that should be unitized (No Default)
// @value {Number}: The pixel value (No Default)
// @baseValue {Number}: The default browser font-size (Default: @remScaleFactor)
.unitize(@property, @value, @baseValue);

// Example
.unitize(margin-top, 20);

// Output
margin-top: 1.25rem // @baseValue was set to 16px

// Optimised unitize function
.unitize-height(@value, @baseValue);
.unitize-width(@value, @baseValue);
.unitize-max-height(@value, @baseValue);
.unitize-max-width(@value, @baseValue);
.unitize-min-width(@value, @baseValue);
.unitize-margin(@topValue, @rightValue, @bottomValue, @leftValue, @baseValue);
.unitize-padding(@topValue, @rightValue, @bottomValue, @leftValue, @baseValue);
.unitize-variable(@value, @baseValue);

Clearfix

Allows the easy integration of the floating-solution method

The clearfix resolves the problem that occurs when two floating elements are nested inside a parent element, then sometimes the parent elements height disappears. Add the .clearfix class to the parent element or the .clearfix() mixin to the styling of your element. 

.clearfix();
